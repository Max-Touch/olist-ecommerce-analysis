# Анализ данных бразильского маркетплейса Olist

## О проекте
Исследование реальных данных бразильского e-commerce маркетплейса Olist за 2016–2018 гг.  
Цель — выявить ключевые проблемы бизнеса: почему клиенты недовольны, где теряется выручка и как сегментировать клиентов.

## Данные
Источник: [Kaggle — Brazilian E-Commerce Public Dataset by Olist](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce)

- ~100 000 заказов за 2016–2018 гг.
- 9 связанных таблиц: заказы, товары, продавцы, клиенты, отзывы, платежи, геолокация
- Реальные данные, анонимизированные Olist

## Стек
- **PostgreSQL** — хранение данных
- **SQL** — агрегации, джойны, CTE, оконные функции
- **Python / pandas** — обработка результатов и вычисление метрик
- **seaborn / matplotlib** — визуализация

## Что анализировали
1. Динамика выручки по месяцам
2. Топ категории товаров по выручке
3. Среднее время доставки по штатам
4. Влияние опозданий на оценку клиентов
5. RFM сегментация клиентов (Recency, Frequency, Monetary)
6. Retention Rate
7. Топ и худшие продавцы

## Главные выводы

1. **Рост бизнеса** — выручка стабильно росла в 2016–2018, пик в ноябре 2017 (Black Friday)
2. **Логистические проблемы** — штаты RR, AP, AM показывают наибольшее время доставки из-за сложной инфраструктуры
3. **Опоздание = плохой отзыв** — средняя оценка падает с 4.2 до 2.5 при опоздании доставки
4. **Retention Rate всего 3%** — главная проблема, почти все клиенты не возвращаются (норма 20-30%)
5. **Топ категория** — health_beauty приносит наибольшую выручку
6. **Проблемные продавцы** — ряд продавцов с большим объемом заказов имеют рейтинг ниже 2.5
